# 架构上下文

## 整体结构

- 运行时：`Next.js 16`
- UI：`React 19 + Tailwind CSS 4`
- 部署：`OpenNext + Cloudflare Workers`

## 当前架构原则

- 游戏大厅与单游戏目录共用一个应用壳。
- 每个游戏通过独立路由承载，避免把所有玩法代码耦合到首页。
- 先维持轻量前端架构，不提前引入数据库和后台抽象。

## 扩展方向

- 后续可以在 Cloudflare 体系内补充 D1、KV、R2。
- 每个游戏在变复杂后，应新增独立上下文文档，避免项目背景和游戏实现混在一起。
- 技术方案定稿后进入 `docs/decisions/`，不把讨论过程长期堆在 `progress.md`。

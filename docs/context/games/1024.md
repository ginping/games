# 1024 游戏上下文

## 目标与当前状态

- 当前是项目中的第一款已完成最小可玩版本的游戏，用于验证大厅到独立子路由、游戏逻辑分层和基础交互闭环已经打通。
- 当前已具备真实棋盘逻辑、得分逻辑、最高分持久化，以及人工试玩验收通过的可玩质量。

## 技术结构

- 路由位置：`app/1024/page.tsx`
- 页面职责：承载 `1024` 的页面容器与返回入口。
- 核心逻辑：`app/1024/game-logic.ts`，包含棋盘状态、滑动、合并、胜负判定。
- 交互状态：`app/1024/use-game.ts`，包含键盘输入、触摸滑动、新开局、继续游戏与最高分持久化。

## 后续演进

- 保持当前可玩体验稳定，作为后续小游戏接入的实现参考。
- 如有需要，可继续补动画、音效、排行榜或存档能力。
- 如果需要排行或存档，再考虑对接 Cloudflare 的持久化能力。

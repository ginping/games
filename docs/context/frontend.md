# 前端约束

## 当前约定

- 新增页面优先使用 App Router 目录结构。
- 游戏入口通过 `lib/games.ts` 统一注册，再由首页读取渲染。
- 页面样式遵循现有 Tailwind 写法，不提前做组件库抽象。

## 目录约定

- 首页入口页位于 `app/page.tsx`。
- 单游戏页面位于 `app/<slug>/page.tsx`。
- 如果某个游戏需要独立组件或状态，可在该游戏目录下继续扩展，而不是先做全局抽象。

## 文档约定

- 涉及页面结构、交互模式、路由策略的稳定结论，优先写入本文件或对应游戏上下文。
- 与一次性迭代强绑定的短期事项，写入 `docs/progress.md` 而不是长期上下文。

# 项目背景

## 项目目标

- 用一个 Next.js 应用承载多个可独立访问的 Web 游戏。
- 首页作为统一入口，每个游戏保留自己的独立目录和上下文。
- 先完成前端骨架，再逐步接入真实玩法、部署能力和后续服务端扩展。

## 当前边界

- 当前已有首页大厅与 `/1024` 示例路由。
- 当前不包含登录、排行榜、数据库、对象存储、后台管理。
- 当前部署目标为 Cloudflare Workers，通过 OpenNext 输出。

## 核心术语

- `Games Center`：项目整体入口页面。
- `Game Entry`：注册在 `lib/games.ts` 中的游戏入口元数据。
- `Game Route`：位于 `app/<slug>/` 的独立游戏目录。

## 目录职责

- `app/`：应用路由与页面。
- `lib/`：共享数据结构与入口注册表。
- `docs/`：项目背景、决策、进展、变更归档。
- `.agents/skills/project-doc-maintainer/`：提交前文档维护技能。
- `scripts/docs/`：文档判定、同步、归档、守卫脚本。
